DNS::
: Before configure DNS make sure to stop the below services
      SELINUX and IPTABLES with 'setenforce 0' and 'service iptables stop'
    1.Configure fully qualified donmain name
      #vi /etc/hosts
         Remove the existing data and enter the below data
           SriRama.example.com 192.168.246.128(Hear SriRama hotsname and example.com is domain name and IP address current system ip)
         Save and exit(:wq!)
      #vi etc/sysconfig/network
         Enter the below data
          NETWORKING=yes
          HOSTNAME=SriRama.example.com       
         Save and exit(:wq!)
    2.Reboot the server
      #reboot
      Now check the host name by click below command
      #hostname (Now hotname will show like SriRama.example.com)
    3.Verfy the packages are installed
      #rpm -qa bind*
    4.Install the packages s
      #yum install bind* caching-name* -y
    5.Edit the DNS configuration
      #vi /etc/named.conf
         Change the 11&17th lines like below.To asign numbers for each like press ':se nu' and enter
            11th line-> listen-on port 53 {192.168.246.128; };(IP is the current system IP Address)
            17th line-> allow-query     { 192.168.246.128; };
    6.Edit the zoning configuration file
      # vi /etc/named.rfc1912.zones
        Remove the content in the file and enter(copy) below data into file
         // named.rfc1912.zones:
         //
         // Provided by Red Hat caching-nameserver package
         //
         // ISC BIND named zone configuration for zones recommended by
         // RFC 1912 section 4.1 : localhost TLDs and address zones
         // and http://www.ietf.org/internet-drafts/draft-ietf-dnsop-default-local-zones-02.txt
         // (c)2007 R W Franks
         //
         // See /usr/share/doc/bind*/sample/ for example named configuration files.
         //
         zone "localhost" IN {
                 type master;
                 file "named.localhost";
                 allow-update { none; };
         };
       
               
         zone "example.com" IN {              
                 type master;
                 file "kt.flz";
                 allow-update { none; };
         };
  
         zone "246.168.192.in-addr.arpa" IN {
                 type master;
                 file "kt.rlz";
                 allow-update { none; };
         };

       Note:example.com is domain name and kt.flz and kt.rlz are created hear for copy the data from named.localhost file.
     7.#cd /var/named      Create Zoning configuration fils
       #cp named.localhost kt.flz
       #cp named.localhost kt.rlz
     8.give the full permissions for the kt.flz&kt.rlz files
       #chmod 777 kt.flz
       #chmod 777 kt.rlz
     9.Edit the forward zoning configuration files
       #vi kt.flz
         Remove the data in file and copy the below data
          $TTL 86400
          @       IN SOA  @ SriRama.example.com. (
                                                  4       ; serial
                                                  16      ; refresh
                                                  17      ; retry
                                                  18      ; expire
                                                  3 )     ; minimum
                  NS      SriRama.example.com.
          SriRama A       192.168.246.128
       #vi kt.rlz 
         Remove the data in file and copy the below data 
          $TTL 86400
          @       IN SOA  @ SriRama.example.com. (
                                                  0       ; serial
                                                  1D      ; refresh
                                                  1H      ; retry
                                                  1W      ; expire
                                                  3H )    ; minimum
                  NS      SriRama.example.com.
          SriRama A       192.168.246.128
          128     PTR     SriRama.example.com
          Note:dott(.) must be after .com and 128 is the last number in ip
        10.Validate zoning configuration
           #named-checkzone example.com kt.flz
           #named-checkzone example.com kt.rlz
        11.Modify resolve the configuration file
           #vi /etc/resolv.conf
            Remove the data in file and copy the below data
             # Generated by NetworkManager
             domain localdomain
